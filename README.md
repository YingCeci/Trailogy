# hikeCompanion — Gemma 4 + Kokoro on iPhone

An iOS app that runs **Gemma 4 E2B** (Google's edge LLM) and
**Kokoro TTS** (mlalma's MLX port) entirely on-device. Ask a question,
Gemma streams a response, Kokoro speaks it aloud. v0 building block for
the full hike-companion park-classroom hackathon project.

## Requirements

- macOS with Xcode 26+
- **iPhone 15 Pro / 16 Pro / 17 Pro or newer** running iOS 18.0+
  (8 GB RAM minimum — Gemma 4 INT4 is ~3.5 GB resident)
- The iOS Simulator does **not** support MLX
- Apple Developer account (free Apple ID is fine for sideloading)
- ~5 GB free disk space for model weights

## Quick start

```bash
git clone https://github.com/YingCeci/hikeCompanion.git
cd hikeCompanion

# 1) Kokoro TTS model + voices (~630 MB)
bash scripts/fetch-models.sh

# 2) Gemma 4 E2B INT4 (~3.5 GB)  — primary repo
bash scripts/fetch-gemma.sh
#    or, if the primary has the PLE quantization bug:
# bash scripts/fetch-gemma.sh --backup

# 3) Generate the Xcode project
bash scripts/generate-project.sh
```

Then in Xcode:

1. Open `HikeCompanion.xcodeproj`
2. Select the project (top of left sidebar) → TARGETS → HikeCompanion → **Signing & Capabilities** → set Team to your Apple ID
3. Plug in iPhone, pick it in the device dropdown (top toolbar)
4. **⌘R**

First launch initialises MLX, which takes ~10–30 s. Subsequent synthesises
are fast — expect ~3× realtime on iPhone 13 Pro after warmup, faster on
newer Pro devices.

## Why mlalma and not mattmireles?

We initially scaffolded against mattmireles/kokoro-coreml because it routes
inference to the Apple Neural Engine (vs. mlalma's MLX which uses GPU). On
paper, ANE is the right architectural choice — it doesn't compete with
Gemma's GPU/CPU usage. But end-to-end on iPhone:

- mattmireles' Swift API takes pre-tokenized `input_ids`, not text — there's
  no Swift G2P/phonemizer in the upstream.
- The public HF model artifacts (`mattmireles/kokoro-coreml`) ship a
  different set of `.mlpackage` files than what the Swift code expects.
  The right files have to be regenerated by running PyTorch → Core ML
  conversion locally (~30–60 min).
- For a hackathon timeline, that's too much yak-shaving for a first run.

mlalma ships text → audio out of the box, with MisakiSwift G2P bundled,
verified at 3.3× realtime on iPhone 13 Pro. So the GPU contention with
Gemma is a real concern but a manageable one — we mitigate by pipelining
TTS at sentence granularity (synth a sentence, play it, while Gemma
continues thinking on the next answer) rather than streaming TTS over
Gemma's token output.

## Architecture

```
[Mac, one-time setup]
  scripts/fetch-models.sh
    └─ curl https://github.com/mlalma/KokoroTestApp/raw/main/Resources/{kokoro-v1_0.safetensors, voices.npz}
       → HikeCompanion/Resources/Models/

[iPhone, every run]
  HikeCompanionApp.swift  →  ContentView (TextField + voice picker)  →  ValidationRunner
                                                                         ├─ KokoroTTS(modelPath:) once at launch
                                                                         ├─ NpyzReader.read(...) → [voiceName: MLXArray]
                                                                         └─ tts.generateAudio(voice:language:text:) per tap
                                                                            → AVAudioEngine playback
                                                                            → AudioUtils.writeWavFile to Documents/
                                                                            → UIActivityViewController for AirDrop
```

## Layout

```
hikeCompanion/
├── README.md                           # this file
├── project.yml                         # xcodegen spec → .xcodeproj
├── scripts/
│   ├── fetch-models.sh                 # downloads .safetensors + .npz
│   └── generate-project.sh             # xcodegen wrapper
└── HikeCompanion/
    ├── HikeCompanionApp.swift          # @main
    ├── ContentView.swift               # TextField, voice picker, run button
    ├── ValidationRunner.swift          # KokoroTTS wrapper, timing, playback
    ├── Info.plist
    ├── Assets.xcassets/
    └── Resources/
        └── Models/                     # safetensors + npz (gitignored)
```

## Cost / size

| | |
|---|---|
| App bundle (debug) | ~640 MB (mostly the safetensors) |
| First-build time | ~1–2 min on M-series Mac |
| First-launch latency | 10–30 s (MLX graph compile) |
| Subsequent synth latency | ~1–3 s for a 5–10 s utterance on iPhone 15 Pro |
